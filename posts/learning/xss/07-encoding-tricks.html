Encoding Tricks

What encoding bypass is

We change the byte representation so the filter sees benign text while the browser sees executable code after auto-decoding.

Why it happens

Multiple decode layers: URL parser, HTML parser, CSS parser, JS parser, Unicode normaliser. Each layer has its own alphabet.

Where you’ll meet it

WAF, email HTML cleaners, Markdown renderers, CMS “safe HTML” modules, PDF generators.

Impact

One encoding trick resurrects a blocked payload → full chain, often worm-able.

Mini case-study

2020 Outlook.com UTF-7 BOM inside EXIF → stored XSS in avatar → 400 M mailboxes affected.

Payloads

// Force older IE to decode as UTF-7
+ADw-script+AD4-alert(1)+ADw-/script+AD4-
// Framework decodes once, browser decodes again
%2522 → %22 → "
// : visually identical to colon
javascript:alert(1)
// U+0061 == a, but filter checks raw bytes
javascript:
// \26 == & → reconstruct tag
\000026 00003c script\00003e
