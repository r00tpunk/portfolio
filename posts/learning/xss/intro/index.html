What is XSS?

Cross-Site Scripting (XSS) is a client-side code-injection attack that lets an attacker execute malicious JavaScript in another user's browser.

Main XSS Flavours

  • Reflected (Non-persistent) – payload in URL/form, executed immediately.
  • Stored (Persistent) – payload saved on server (DB, comment, profile), executed later for every visitor.
  • DOM-based – pure client-side; JavaScript reads dangerous data from document.location / hash / referrer and writes it to page without sanitising.

Quick Test Payloads

Defence Cheat-Sheet

  1. Encode on output – HTML, JS, CSS, URL contexts need different escaping rules.
  2. Content-Security-Policy – lock down sources, disable inline scripts.
  3. Input validation – whitelist acceptable format, length, charset.
  4. Modern frameworks – React/Vue/Svelte auto-escape; still audit dangerouslySetInnerHTML / v-html.
  5. HttpOnly + Secure + SameSite cookies – stop cookie theft via XSS.

✅ No user input is trusted until it’s encoded for the context it lands in.
